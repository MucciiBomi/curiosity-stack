# Attribution Requirements

Curiosity Stack is MIT licensed. Forks, modifications, and derivatives
are welcome — attribution is required.

## Mandatory attribution on all derivatives

If you fork, modify, or build upon Curiosity Stack, you MUST include
all of the following in any output, export, or published artifact:

### 1. Clickable credit link — required on every output
The attribution credit must be a clickable hyperlink. Text-only credit
is not sufficient.

Required format:
```
Built with Curiosity Stack → curiositystack.app
```

The text "Curiosity Stack" or "curiositystack.app" must link to:
```
https://curiositystack.app
```

A non-clickable text mention does not satisfy this requirement.

### 2. README credit — required in every forked repository
Your repository README must include:

```markdown
## Attribution
Built on [Curiosity Stack](https://curiositystack.app) by 
[Ameya Pimpalgaonkar](https://x.com/Finstor85).
Original framework: [github.com/ameya85/curiosity-stack](https://github.com/ameya85/curiosity-stack)
```

### 3. LICENSE file — must remain intact
Do not modify or remove the LICENSE file. It is the legal basis for
your right to use this code.

---

## What constitutes proper attribution

✅ "Built with Curiosity Stack" linking to curiositystack.app
✅ README credit with hyperlink to original repo
✅ LICENSE file present and unmodified

❌ "Built with Curiosity Stack" as plain non-clickable text
❌ Attribution buried in source code comments only
❌ Removing the attribution entirely
❌ Replacing attribution with a different product name

---

## Reporting attribution violations

If you see a fork or derivative that does not properly attribute
Curiosity Stack, please report it:

- Open an issue at github.com/ameya85/curiosity-stack/issues
- Or contact @Finstor85 on X / Twitter
